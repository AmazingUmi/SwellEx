# HDF5 Datasets and Split Strategies

## Current Layout

MATLAB now has two dataset-generation routes:

```text
scripts_matlab/RBD_method/Signals_Segmentation.m
scripts_matlab/ELM_method/Signals_Segmentation.m
```

Both read original data from:

```text
Origindata/
  events/
  positions/
  environments/
  source/
```

Both write train/test HDF5 files under:

```text
outputs/Datasets/<split_strategy>/
```

The common metadata fields are:

```text
/y_range_km                range label in km
/valid_sample              valid-label mask
/time/window_start_s
/time/window_center_s
/time/window_stop_s
/time/sample_start_idx
/time/sample_stop_idx
/frequency/freq_hz
/array/depth_m
/array/element_idx
/range/range_time_s
/range/range_km_raw
/split/source_segment_idx
```

## RBD Dataset

Generated by:

```text
scripts_matlab/RBD_method/Signals_Segmentation.m
```

Feature layout:

```text
/X: [sample, element, frequency, real_imag]
```

Meaning:

```text
X(:,:,:,1) = real(green_freq)
X(:,:,:,2) = imag(green_freq)
```

Additional RBD diagnostics:

```text
/rbd/theta_vec_rad
/rbd/theta_best_rad
/rbd/num_selected_angles
/rbd/theta_selected_rad
/rbd/selected_beam_power
/rbd/beam_power
/rbd/signal_freq_scale
```

Python route:

```text
scripts_py/RBD_method/Network_main.py
```

PyTorch input:

```text
[batch, 2, element, frequency]
```

## ELM Pairwise-Ratio Dataset

Generated by:

```text
scripts_matlab/ELM_method/Signals_Segmentation.m
```

Feature layout:

```text
/X: [sample, numerator_element, denominator_element, frequency, real_imag]
```

Meaning:

```text
ratio_freq(i,j,f) = FFT(element_i,f) / FFT(element_j,f)
X(:,:,:,:,1) = real(ratio_freq)
X(:,:,:,:,2) = imag(ratio_freq)
```

The MATLAB implementation uses the stable equivalent:

```text
FFT_i * conj(FFT_j) / (abs(FFT_j)^2 + denominator_floor)
```

The ELM script can optionally keep only Mel-spaced FFT bins while preserving the
complex ratio values. This selection maps Mel-spaced center frequencies to the
nearest one-sided FFT bins; it does not apply triangular Mel energy weighting.

Python route:

```text
scripts_py/ELM_method/Network_main.py
```

Current Python loader flattens the element-pair grid:

```text
[sample, N, N, F, 2] -> [batch, 2, N*N, F]
```

This keeps the first ELM baseline compatible with 2-D CNN and ResNet models.

## Split Strategies

Common split helpers live in:

```text
scripts_matlab/function/split_function/
```

Current helper prefix:

```text
SPL_*
```

Available split strategies:

- `periodic`: deterministic interleaving controlled by `train_test_ratio`.
- `Range_nearby`: find the minimum range point, take symmetric windows on both
  sides, and use one side for training and the other for testing.

Example:

```matlab
split_strategy = "Range_nearby";
split_options.half_duration_s = 800;
split_options.gap_s = 15;
split_options.train_side = "after";  % or "before"
```

## Training Examples

RBD:

```bash
python3 scripts_py/RBD_method/Network_main.py train \
  --model complex_cnn_range \
  --data Range_nearby_after_800s_gap15s_test_no_beamformer
```

RBD with physical-error loss:

```bash
python3 scripts_py/RBD_method/Network_main.py train \
  --model complex_cnn_range \
  --data Range_nearby_after_800s_gap15s_test_no_beamformer \
  --loss-space km --huber-beta 0.5
```

ELM:

```bash
python3 scripts_py/ELM_method/Network_main.py train \
  --model elm_complex_cnn_range \
  --data Range_nearby_after_800s_gap15s_elm_pairwise_ratio
```

ELM with physical-error loss:

```bash
python3 scripts_py/ELM_method/Network_main.py train \
  --model elm_complex_cnn_range \
  --data Range_nearby_after_800s_gap15s_elm_pairwise_ratio \
  --loss-space km --huber-beta 0.5
```

Prediction uses the same method-specific entry point with `predict`.
