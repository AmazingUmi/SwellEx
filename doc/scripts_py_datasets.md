# HDF5 Datasets and Split Strategies

## Current Layout

MATLAB now has three dataset-generation routes:

```text
scripts_matlab/RBD_method/Signals_Segmentation.m
scripts_matlab/ELM_method/Signals_Segmentation.m
scripts_matlab/SCM_method/Signals_Segmentation.m
```

All three read original data from:

```text
Origindata/
  events/
  positions/
  environments/
  source/
```

All three write train/test HDF5 files under:

```text
outputs/Datasets/<split_strategy>/
```

The common metadata fields are:

```text
/y_range_km                range label in km
/valid_sample              valid-label mask
/time/window_start_s
/time/window_center_s
/time/window_stop_s
/time/sample_start_idx
/time/sample_stop_idx
/frequency/freq_hz
/array/depth_m
/array/element_idx
/range/range_time_s
/range/range_km_raw
/split/source_segment_idx
```

## RBD Dataset

Generated by:

```text
scripts_matlab/RBD_method/Signals_Segmentation.m
```

Feature layout:

```text
/X: [sample, element, frequency, real_imag]
```

Meaning:

```text
X(:,:,:,1) = real(green_freq)
X(:,:,:,2) = imag(green_freq)
```

```text
/rbd/theta_vec_rad
/rbd/theta_best_rad
/rbd/num_selected_angles
/rbd/theta_selected_rad
/rbd/selected_beam_power
/rbd/beam_power
/rbd/signal_freq_scale
```

```text

```

PyTorch input:

```text
[batch, 2, element, frequency]
```

## ELM Least-Squares Ratio Dataset

Generated by:

```text
scripts_matlab/ELM_method/Signals_Segmentation.m
```

Default feature layout:

```text
/X: [sample, pair, frequency, real_imag]
/pair/numerator_element_idx: [pair, 1]
/pair/denominator_element_idx: [pair, 1]
```

Meaning:

```text
pair contains strict upper-triangle element pairs with i < j
X(:,:,:,1) = real(ratio_freq)
X(:,:,:,2) = imag(ratio_freq)
```

The current MATLAB implementation estimates element ratios by least squares
across `Ns` consecutive snapshots:

```text
ratio_freq(i,j,f)
  = sum_s FFT_i,s(f) * conj(FFT_j,s(f))
    / (sum_s abs(FFT_j,s(f))^2 + denominator_floor)
```

ELM segmentation is controlled by `segment_duration_s`,
`num_snapshots_per_segment`, and `snapshot_overlap_count`. For example,
`Ns = 4` and overlap `3` creates 1-4 s, 2-5 s, 3-6 s, ... windows. The HDF5
`window_*` fields and `/y_range_km` refer to the combined `Ns`-snapshot window,
and the generated dataset tag records the actual snapshot settings.

ELM and SCM share the same configurable frequency-bin selection helper:

```text
scripts_matlab/function/dataset_function/DS_select_frequency_bins.m
```

The MATLAB parameter `frequency_selection_modes` accepts one mode or a vector of
modes:

```matlab
frequency_selection_modes = "mel";
frequency_selection_modes = "deep";
frequency_selection_modes = "shallow";
frequency_selection_modes = ["deep", "shallow"];
frequency_selection_modes = "adapt";
frequency_selection_modes = ["mel", "deep", "adapt"];
```

Supported choices:

- `full`: keep all one-sided FFT bins.
- `mel`: keep nearest FFT bins to Mel-spaced center frequencies.
- `deep`: keep `[49 64 79 94 112 130 148 166 201 235 283 338 388]` Hz.
- `shallow`: keep `[109 127 145 163 198 232 280 335 385]` Hz.
- `adapt`: estimate dataset-level average spectral power from valid candidate
  windows and keep the strongest `adapt_num_bins` bins.

When several modes are combined, duplicated FFT bins are removed and the final
frequency axis is sorted by increasing FFT bin. `adapt` is dataset-level, not
per-window, so the CNN frequency dimension keeps a fixed physical meaning.

Python route:

```text
scripts_py/ELM_method/Network_main.py
```

Current Python loader reads the pair vector directly:

```text
[sample, P, F, 2] -> [batch, 2, P, F]
P = N * (N - 1) / 2
```

## SCM Dataset

Generated by:

```text
scripts_matlab/SCM_method/Signals_Segmentation.m
```

Feature layout:

```text
/X: [sample, pair, frequency, real_imag]
/pair/numerator_element_idx: [pair, 1]
/pair/denominator_element_idx: [pair, 1]
```

Meaning:

```text
C_q = mean_s((x_s / ||x_s||) (x_s / ||x_s||)^H)
pair contains upper-triangle entries with diagonal, i <= j
P = N * (N + 1) / 2
X(:,:,:,1) = real(C_q pair-vector)
X(:,:,:,2) = imag(C_q pair-vector)
```

SCM segmentation uses the same snapshot parameters as ELM:
`segment_duration_s`, `num_snapshots_per_segment`, and
`snapshot_overlap_count`. For example, `Ns = 4` and overlap `3` averages
1-4 s, 2-5 s, ... windows.

## Dataset Variant Tags

ELM and SCM dataset scripts automatically build a variant tag from the feature
type, frequency selection, snapshot length, `Ns`, and overlap:

```text
elm_pairwise_lsr_upper_mel64_snap1s_ns4_ov3
elm_pairwise_lsr_upper_deep_shallow_snap1s_ns4_ov3
elm_pairwise_lsr_upper_mel64_adapt16_snap1s_ns4_ov3
scm_upper_diag_mel64_snap1s_ns4_ov3
scm_upper_diag_deep_shallow_snap1s_ns4_ov3
```

Set `manual_dataset_variant_tag` in the MATLAB script to append an extra suffix
after the automatic tag:

```matlab
manual_dataset_variant_tag = "runA";
```

This produces names such as:

```text
periodic_4_1_elm_pairwise_lsr_upper_mel64_snap1s_ns4_ov3_runA
```

After writing the dataset, the ELM and SCM scripts print the dataset name plus
recommended train and predict commands to the MATLAB Command Window.

Python route:

```text
scripts_py/SCM_method/Network_main.py
```

## Split Strategies

Common split helpers live in:

```text
scripts_matlab/function/split_function/
```

Current helper prefix:

```text
SPL_*
```

Available split strategies:

- `periodic`: deterministic interleaving controlled by `train_test_ratio`.
- `Range_nearby`: find the minimum range point, take symmetric windows on both
  sides, and use one side for training and the other for testing.
  Optional `split_options.test_step_s` subsamples only the test side; for SCM,
  `test_step_s = 4` gives non-overlapping 4 s test spacing and roughly
  `train:test = 4:1`.

Example:

```matlab
split_strategy = "Range_nearby";
split_options.half_duration_s = 800;
split_options.gap_s = 15;
split_options.train_side = "after";  % or "before"
split_options.test_step_s = 4;        % optional, test side only
```

## Training Examples

RBD:

```bash
python3 scripts_py/RBD_method/Network_main.py train \
  --model complex_cnn_range \
  --data Range_nearby_after_800s_gap15s_test_no_beamformer
```

RBD with physical-error loss:

```bash
python3 scripts_py/RBD_method/Network_main.py train \
  --model complex_cnn_range \
  --data Range_nearby_after_800s_gap15s_test_no_beamformer \
  --loss-space km --huber-beta 0.5
```

ELM:

```bash
python3 scripts_py/ELM_method/Network_main.py train \
  --model elm_complex_cnn_range \
  --data <elm_dataset_name>
```

ELM with physical-error loss:

```bash
python3 scripts_py/ELM_method/Network_main.py train \
  --model elm_complex_cnn_range \
  --data <elm_dataset_name> \
  --loss-space km --huber-beta 0.5
```

SCM:

```bash
python3 scripts_py/SCM_method/Network_main.py train \
  --model scm_complex_cnn_range \
  --data <scm_dataset_name>
```

Prediction uses the same method-specific entry point with `predict`.
